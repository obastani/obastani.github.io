Research Interests

In the last few decades, a number of program analysis tools have been developed to help identify security vulnerabilities and improve software quality. Many of these tools rely on access to program source code, an assumption that often fails in practice; for example, calls to native functions, code in third-party libraries, or dynamically loaded code. Sometimes, the source code is available but the specification is missing.

I am broadly interested in the problem of analyzing such partial programs, where part (or all) of the program source code (or specification) is unavailable. My approach is to infer program properties based on auxiliary information such as usage or execution traces of the missing code. To do so, I have leveraged a range of techniques, including machine learning, optimization, artificial intelligence, and program synthesis.
